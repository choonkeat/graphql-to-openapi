This schema demonstrates REST pattern detection. When the converter finds both list operations (users: [User!]!) AND create mutations (createUser), it consolidates them into proper RESTful routes with HTTP methods.

Pattern detection rules:

  1. Must have BOTH {resource}s: [T] + create{Resource}(...) to trigger
  2. Strict matching - no fuzzy matching (e.g., addUser won't match)
  3. Additional operations (get, update, delete) are optional but recognized
